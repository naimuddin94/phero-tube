Interview Questions

Discuss the scope of var, let, and const

var

  • Function scope: Variables are accessible throughout the entire function in which they are declared.
  • Hoisting: Variables are hoisted to the top of their containing function or script, but only the declaration is hoisted, not the initialization.

let

  • Block scope: Variables are only accessible within the block (e.g., if, for) in which they are defined.
  • Hoisting: Variables are hoisted but have a "temporal dead zone" where they cannot be accessed before their declaration.

const

  • Block scope: Variables are only accessible within the block (e.g., if, for) in which they are defined.
  • Immutability: Variables declared with const cannot be reassigned new values, but for objects and arrays, their properties or elements can be modified.
Tell us the use cases of null and undefined

null

null is a value that represents the explicit absence of any object value or a value that is intentionally missing or unknown. It is often used to indicate that a variable or property has been deliberately set to "no value."

undefined

undefined is the default value of uninitialized variables and function parameters in JavaScript. It indicates that a variable or property has been declared but has not yet been assigned any value.

What do you mean by REST API?

REST API, which stands for Representational State Transfer Application Programming Interface, is a set of architectural constraints and principles for designing networked applications. It is an approach to building web services that are scalable, stateless, and easy to understand and use.
